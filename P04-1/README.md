# • 최대공약수(GCD) 시간복잡도 보고서 •

## 시간 복잡도 (Time Complexity): $O(\log N)$
- **분석:** `Get_GCD` 함수 내의 `while` 루프는 나머지 연산(`num1 % num2`)을 수행합니다. 이 과정에서 연산의 대상이 되는 숫자의 크기가 매 단계마다 최소 절반 이하로 감소하는 수학적 특성이 있습니다.
- **결론:** 처리해야 할 데이터의 크기가 아무리 커지더라도 연산 횟수는 숫자의 비트 수(자릿수)에 비례하여 매우 천천히 증가합니다. 따라서 전체 시간 복잡도는 최악의 경우에도 **$O(\log(\min(num1, num2)))$**, 즉 $Big-O$ 는 **$O(\log N)$** 의 우수한 성능을 보입니다.
- **참고:** `main` 함수의 `while(1)` 루프는 사용자가 올바른 값을 입력할 때까지 반복되는 루프이므로, 핵심 알고리즘 자체의 시간 복잡도 계산에서는 제외하는 것이 일반적입니다.
